SingleSnapshotContainer

Something that can hold a single “snapshot” of an object. This is to be passed to models that need to be persisted in a simple load/save fashion.

  • load(_:)

    Asynchronously loads a snapshot from the container. A nil is returned if no previous state was stored.

  • loadSync(_:)

    The synchronous version of load(). The latter is preferred for large object, but this one might be easier to use in the existing code.

  • save(_:)

    Schedules to save the given snapshot replacing anything previously saved. Note that this might be performed asynchronously and lazily (e.g. only when the app goes to background), so the snapshot should be able to encode itself on a different queue.

  • clean()

    Clean this container (e.g. remove contents), but keep it alive for later use. If you don’t need to keep the container around, use SnapshotStorage.removeContainerForKey(_:) instead.

  • cleanSync()

    The synchronous version of clean().
